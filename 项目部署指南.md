# 🚀 项目部署指南 - 简历展示版

本文档指导如何将这个RAG企业知识库项目部署到线上，让HR可以通过链接直接访问。

## 📋 部署方案选择

### 方案一：Streamlit Cloud（推荐⭐ - 免费且简单）

**优点**：
- ✅ 完全免费
- ✅ 部署简单（5分钟）
- ✅ 自动更新（GitHub推送即更新）
- ✅ 支持环境变量配置
- ✅ 提供公开访问链接

**步骤**：

1. **准备GitHub仓库**
   ```bash
   # 确保项目已推送到GitHub
   git init
   git add .
   git commit -m "RAG企业知识库系统"
   git remote add origin https://github.com/你的用户名/你的仓库名.git
   git push -u origin main
   ```

2. **注册Streamlit Cloud**
   - 访问 https://share.streamlit.io/
   - 使用GitHub账号登录
   - 点击 "New app"

3. **配置部署**
   - Repository: 选择你的GitHub仓库
   - Branch: `main` 或 `master`
   - Main file path: `app.py`
   - App URL: 可以自定义（如：`rag-enterprise-qa`）

4. **配置环境变量**
   - 在Streamlit Cloud的Secrets中配置：
     ```
     DASHSCOPE_API_KEY=your_api_key_here
     ```
   - 或者使用测试数据（不依赖API）

5. **部署完成**
   - 部署成功后，你会得到一个公开链接，例如：
     `https://你的应用名.streamlit.app`
   - 这个链接可以直接放在简历上！

**优点**：
- ✅ 免费

- ✅ 社区友好

**步骤**：

4. 上传代码

## 🔒 安全注意事项

1. **API密钥保护**
   - 不要将API密钥提交到GitHub
   - 使用环境变量或Secrets管理
   - 考虑使用免费的测试API

2. **数据隐私**
   - 确保上传的数据不涉及敏感信息
   - 使用公开可用的测试数据

3. **访问控制**
   - Streamlit Cloud免费版为公开访问
   - 如需私有部署，考虑付费方案

## 📞 常见问题

### Q: 部署失败怎么办？
A: 
1. 检查requirements.txt是否完整
2. 检查Python版本兼容性（建议3.8+）
3. 查看Streamlit Cloud的日志

### Q: API调用费用怎么办？
A: 
1. 使用DashScope的免费额度
2. 或创建演示版本，使用预处理结果

### Q: 数据文件太大怎么办？
A: 
1. 使用Git LFS
2. 或使用云端存储（如AWS S3）
3. 或仅使用测试数据集
